## 有限状态机

有限状态机（Finite State Machine, FSM）是一种数学模型,用于描述具有有限个状态的系统.它可以在任意时刻处于一个状态,并根据输入的事件或条件在这些状态之间进行转换.有限状态机广泛应用于计算机科学、自动化控制、游戏开发等领域.

有限状态机的基本组成部分包括：

1. **状态（States）**：系统可以处于的不同情况或配置.每个状态代表系统在某一时刻的特定条件.

2. **输入（Inputs）**：触发状态转换的事件或条件.输入可以是外部事件（如用户操作）或内部条件（如计时器到期）.

3. **转换（Transitions）**：描述从一个状态到另一个状态的变化规则.每个转换通常与特定的输入相关联.

4. **初始状态（Initial State）**：系统开始时的状态.

5. **接受状态（Accept States）**：在某些应用中,特定的状态被定义为接受状态,表示系统完成了某个任务或达到了某个条件.

有限状态机可以分为两种类型：确定性有限状态机（DFA）和非确定性有限状态机（NFA）.DFA在每个状态下对于每个输入都有唯一的下一个状态,而NFA则允许在某些状态下对于同一输入有多个可能的下一个状态[1](https://brilliant.org/wiki/finite-state-machines/)[2](https://medium.com/@mlbors/what-is-a-finite-state-machine-6d8dec727e2c).

有限状态机的一个经典例子是交通信号灯,它在红灯、黄灯和绿灯之间根据时间或交通情况进行切换.每种灯光状态代表一个状态,而时间的变化或交通流量的变化则是输入,决定了状态的转换[3](https://eng.libretexts.org/Under_Construction/Book%3A_Discrete_Structures/09%3A_Finite-State_Automata/9.01%3A_Introduction/9.1.01%3A_Finite-State_Machine_Overview).

## markdown中使用数学公式

- 安装插件

```bash
pnpm --filter @notes/docs i remark-math rehype-katex -r
```

```js
import remarkToc from "remark-toc";
import remarkMath from "remark-math";
import { defineConfig } from "astro/config";
//***
export default defineConfig({

    // ***
    markdown: {
        remarkPlugins: [
            remarkMath,
        ],
        rehypePlugins: [
            rehypeKatex,

         ],
})
```
